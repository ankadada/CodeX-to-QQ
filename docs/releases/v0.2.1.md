# CodeX-to-QQ v0.2.1

这是一版面向“长期日用”的体验迭代，重点不是堆更多命令，而是把 QQ 里真实会遇到的细节补顺：纯文本模式、确认流、状态反馈、workspace / session 手感，以及网关与服务脚本的稳定性。

## 这版值得关注的点

### 1. 纯文本 QQ 会话更好用了

- 当 QQ 当前会话不支持自定义按钮时，会自动降级为纯文本模式
- `/help` 会自动切成更适合 QQ 手打的短命令菜单
- 新增 `/help quick`，适合快速上手或给新用户看
- 纯文本模式下支持数字快捷菜单，可直接回复 `1`、`2`、`3`

### 2. 危险操作确认流更稳

- `/rollback all`
- `/mode dangerous`

这些高风险操作会先进入确认流。

如果确认提示被聊天刷走了，现在可以直接：

- 发 `/confirm-action list` 查看当前待确认项
- 发 `/confirm-action latest confirm` 直接确认最新一条

同时，查看确认详情本身不会再误消耗那条确认。

### 3. 状态和诊断更像“操作面板”

- `/status` 不再只是静态信息，而会给出下一步建议
- `/diag` / `/doctor` 会提示当前更适合执行的命令
- 会根据是否在运行、是否排队、是否纯文本、是否有待确认操作给出不同提示

这让 QQ 里直接排障、续聊、切换工作区更顺手。

### 4. workspace / session 工作流更完整

- `/rename`、`/pin`、`/fork` 让会话更适合长期保留
- `/workspace recent` 可直接回数字切换最近项目目录
- `/repo`、`/branch`、`/diff`、`/commit`、`/rollback` 组成轻量 Git 工作流
- `/changed`、`/patch`、`/open`、`/export diff` 方便直接在 QQ 里查看或导出结果

### 5. 稳定性和发布工程同步补齐

- 网关 heartbeat / reconnect / repeated `4009` 处理更稳
- 新增 `restart` / `uninstall` 服务脚本
- `doctor` 输出更完整
- 自动化测试继续补齐
- 仓库补上 GitHub Actions CI，首页 CI 徽章可用

## 适合谁升级

如果你现在是：

- 自己长期用 QQ 连 Codex
- 经常在手机 QQ 里操作
- 会频繁切 workspace / session
- 希望在纯文本场景下也尽量顺手

那 `v0.2.1` 会比首发版明显更稳、更顺。

## 升级方式

```bash
git pull
npm install
npm run ci
npm run service:restart:launchd
```

Linux 用户则可使用：

```bash
npm run service:restart:systemd
```

## 说明

- 这版仍然默认面向自建、可信使用场景
- `dangerous` 模式依然只建议在你完全信任的环境下开启
- OCR 更适合截图、报错界面、UI 图，不建议把它当成通用照片识别能力
