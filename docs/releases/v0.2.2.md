# CodeX-to-QQ v0.2.2

这是一个偏“维护体验”和“开源协作体验”的补丁版本，重点是让排障更快、提 issue 更省心、远程协助更高效。

## 这版更新了什么

### 1. `doctor` 支持 JSON 输出

现在除了：

```bash
npm run doctor
```

还可以直接输出机器可读诊断：

```bash
npm run doctor:json
```

或者：

```bash
node scripts/doctor.mjs --json
```

适合：

- 发 issue 时附带环境信息
- 后续做自动化巡检
- 多机部署时统一比对诊断结果

### 2. `doctor` 会直接告诉你下一步做什么

如果检查失败，不再只是给一串失败项，而会附带更可执行的建议，比如：

- 去补 `.env`
- 检查 QQ bot 凭证
- 安装 Codex CLI
- 修复目录权限
- 安装 / 检查 `launchd` 或 `systemd`
- 调整 OCR 配置

### 3. 故障排查文档补齐

新增：

- `docs/TROUBLESHOOTING.md`
- `docs/TROUBLESHOOTING_CN.md`

覆盖了这些常见问题：

- 机器人完全不回复
- 某些聊天能用、某些不能
- QQ 按钮显示不全
- 危险操作确认提示找不到
- 私聊总是续着同一会话
- gateway 经常重连
- OCR 看起来没生效
- 前台正常、后台服务不稳定

### 4. Issue 模板更容易给出有效信息

现在 bug report 会明确提醒提交：

- `npm run doctor`
- `npm run doctor:json`
- `npm run service:status`
- `/diag`

这样后续排障效率会高很多。

## 适合升级的人

如果你已经开始：

- 长期自建使用
- 在不同机器上跑
- 准备让别人来提 issue / 一起贡献

那这版会更省沟通成本。

## 升级方式

```bash
git pull
npm install
npm run ci
```

后台服务用户再执行：

```bash
npm run service:restart
```
