# CodeX-to-QQ v0.2.0

首个公开版本，目标是把 `QQ -> Codex CLI` 这条链路做成真正适合长期自建使用的桥接服务，而不是一次性的 demo。

## 亮点

- 支持 QQ 单聊 C2C 与群聊 `@bot`
- 每个 QQ 会话独立 Codex session / workspace
- 支持 `/status`、`/progress`、`/stop`、`/new`、`/sessions`
- 支持自动上下文压缩、旧 session 失效自动恢复、自愈重试
- 支持附件下载、图片输入、文档文本抽取
- 支持处理中的进展更新、排队、取消、诊断、统计、审计
- 支持 macOS `launchd` 和 Linux `systemd --user`

## 这版重点做了什么

### 交互体验

- 私聊可主动开启新会话，不再只能永远续同一个会话
- 状态、进展、取消提示做了中文产品化整理
- 增加快捷命令别名：`/stop`、`/state`、`/history`、`/start`
- 短控制消息支持快捷按钮，减少频繁手打命令

### 稳定性

- QQ 网关重连逻辑增强，旧连接退场和 stale socket 处理更稳
- 心跳和 resume 逻辑做了更严格的当前连接绑定
- 被动回复失败时支持更稳的主动消息 fallback

### 可维护性

- 新增 `doctor` 自检脚本
- 新增 CI、测试、Issue / PR 模板
- 新增审计和运行统计命令：`/diag`、`/stats`、`/audit`
- 新增安全说明、贡献文档、变更记录

## 默认定位

这个项目优先面向：

- 个人自建
- 可信小范围使用
- 已经在本机或自有主机上运行 Codex CLI 的用户

对于公网开放、多人共用、强隔离环境，建议先阅读 `SECURITY.md`，再自行做额外收敛和权限控制。

## 已知说明

- `dangerous` 模式适合可信自用，不适合直接对陌生人开放
- QQ 客户端对按钮展示会有端差异
- 进展消息撤回默认关闭，因为 QQ 会明显显示“对方撤回了一条消息”，体感更差
